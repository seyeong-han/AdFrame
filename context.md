# AdFrame context

## Current app

- Production web app scaffold lives at `app-web/` (Next.js 16 App Router, TypeScript, Tailwind v4).
- Main routes:
  - `app/page.tsx` — landing/import screen with Samsung S90F URL prefilled.
  - `app/analysis/page.tsx` — verified/inferred fact review before canvas generation.
  - `app/editor/page.tsx` — tldraw editor shell with assets/copy panels, inspector, export modal, carousel split view, and style preset drawer.
- Top navigation (`app-web/components/AppChrome.tsx`) is now step-focused for hackathon: only `Import`, `Analysis`, and `Editor`; the previous `Open studio` CTA/tab is removed. `AppNav` uses `usePathname()` and `aria-current=\"page\"`; the active step gets the white pill background via `.nav-pill .active`.
- API routes:
  - `app/api/extract/route.ts` — Playwright scrape first, cached fixture fallback, optional OpenAI copy normalization if `OPENAI_API_KEY` exists.
  - `app/api/generate-image/route.ts` — `gpt-image-1` when `OPENAI_API_KEY` exists, local SVG placeholder fallback otherwise.
  - `app/api/remove-background/route.ts` — server-only remove.bg integration. Reads API key from `REMOVE_BG_API_KEY`, `REMOVEBG_API_KEY`, `REMOVEBG_KEY`, or `REMOVE_BG_KEY`; also checks parent `../.env` because the user keeps `REMOVEBG_KEY` at the parent project `.env`.
  - `app/api/extract/route.ts` also extracts product-page design signals (computed fonts, colors, radii, CTA styles) and writes `app-web/generated/design.md` plus `app-web/generated/tokens.source.json`.
  - `app/api/extract/route.ts` filters Samsung PDP images dynamically from the imported URL/SKU and caches remove.bg outputs under `app-web/public/generated/removebg/<sku>/`.
- `app/api/extract/route.ts` now extracts Samsung feature section components before trying background removal. It collects product imagery from `<img>`, `<picture><source srcset>`, lazy `data-*` attributes, CSS `background-image`, and image URLs embedded in page HTML, then builds section assets for Processor/HDR/Motion/Gaming/Soccer plus additional relevant lifestyle/feature visuals. Section crops are cached under `app-web/public/generated/sections/<sku>/`; filenames include a source URL hash so changing candidate order cannot reuse stale crops. `section-components.json` records DOM/OCR-style label inference plus editable recreation metadata.

Analysis/Editor preview them with `<video>`; canvas placement remains image-only for now.
- Demo image manifests are keyed by imported Samsung SKU/cache namespace and loaded by `app/api/extract/route.ts` before live image processing. Current CSVs: `app-web/generated/demo-image-map-qn77s90haexza.csv` (S90H, 15 image assets) and `app-web/generated/demo-image-map-qn77s95hafxza.csv` (S95H, 18 image assets). Each maps PDP URL/model to `hero`, `gallery`, and `section` assets with captions, semantic groups, `bgRemoved`, public paths, local disk paths, and source Samsung URLs for section crops. Use these as stable demo lookup tables when presentation flow should avoid live scraping variance.
- Stable local demo cutouts: `app-web/public/fixtures/samsung-s90f/*.svg`.
- Shared product/export types: `app-web/lib/types.ts`.
- Client state handoff between import, analysis, and editor uses `localStorage` via `app-web/lib/client-store.ts`. SSR/client hydration must use `useExtractionSnapshot()` rather than calling `readExtraction()` in a `useState` initializer; otherwise server fixture HTML can mismatch client live extraction HTML.
- Analysis → Editor handoff is pinned through an approved extraction snapshot (`adframe:approved-extraction`). `AnalysisPage` calls `approveExtraction(extraction)` and `EditorPage` uses `useApprovedExtractionSnapshot()`; the editor also recomposes the tldraw canvas when the approved extraction key changes so the canvas, Assets panel, and Copy panel cannot keep a previous product's assets/copy after approving a new analysis.
- Product extraction can include `designSystem` metadata (`ProductDesignSystem` in `app-web/lib/types.ts`), based on Open Design's portable `design.md` pattern.

## Editor architecture

- Real tldraw is used with custom shape utils in `app-web/lib/tldraw/shapes/adframe-shapes.tsx`.
- Template shapes are parented to the social frame in `app-web/lib/tldraw/templates.ts`; keep this invariant or frame export can omit canvas contents.
- Custom shape types:
  - `ff-canvas-bg` (full-frame dark canvas background; black + one accent radial glow)
  - `ff-glass-card`
  - `ff-glass-text`
  - `ff-cutout-image`
  - `ff-badge`
  - `ff-icon`
- Template composer: `app-web/lib/tldraw/templates.ts`.
- Canonical design system is root `DESIGN.md` — **Apple-Inspired Tile-Based Infographic System (DARK)**. This replaced the earlier light direction. `style.md` is now an implementation-mapping companion to `DESIGN.md` (tone→tldraw table, layout/type/asset rules); when they disagree, `DESIGN.md` wins. The old root `DESIGN.md` (vendored "liquid-glass-agency" open-design skill) was overwritten with the new spec.
- Design posture (DESIGN.md): pure-black export canvas, dark glass tiles (`#111` + subtle white gradient), high-contrast **white** type, product-first hero cutout with one accent glow, accent used ≤2–3× per canvas, 12-col grid (72px margin), tile radius 40px. Tokens live in `app-web/app/globals.css` as `--ig-*` vars.
- Position presets now exist in `app-web/lib/presets.ts`: `cinema-mosaic` preserves the current dark AdFrame story layout, while `apple-infographic` creates a light Apple Watch-reference-style rounded tile board with sparse orange accent, short feature/spec labels, and a true centered hero image. The editor has a `Position preset` selector in `app-web/app/editor/page.tsx`; changing it recomposes the main canvas through `composeMainCanvasTemplate(product, exportPreset, positionPreset)`.
- Apple infographic preset implementation lives in `app-web/lib/tldraw/templates.ts` as `composeAppleInfographicTemplate()`. As of 2026-06-28 it follows the latest user sketch: the top area has a tall left asset spanning the first two rows, with `asset + copy` on the first row and `copy + asset` on the second row; the main row has a left `asset + copy` stack, centered `apple-hero`, and right price/asset stack; the lower row has left/right assets with two centered stacked copy cards; the footer row has left copy, dominant center asset, and compact right asset. It uses a light `ff-canvas-bg` variant plus `paper`/`orange` card tones. Asset organization uses `caption` and `semanticGroup` metadata in the scoring helpers (`describeAsset`, `scoreAssetForSlot`, `pickHeroAsset`, `pickAssetForSlot`); exact or visually similar groups are not reused, and when the sketch has more image slots than unique scenes, duplicate image slots become copy-card fallbacks instead of repeating the same/similar photo.
- Main canvas polish: the old `Verified PDP facts` badge was removed from generated canvases. `composeMainCanvasTemplate()` accepts `{ showPrice }`, and the editor exposes a `Price` checkbox that recomposes the current position preset with price/model tiles included or omitted. Assets/Copy side-panel titles now show live counts (`Assets (n)`, `Copy (n)`).
- Default editor composition: `composeAppleCleanTemplate` inserts `ff-canvas-bg` as the **first child** of the export frame, then a hero cutout + ambient section panel + headline/subtitle + optional price/model badge + a 3-tile feature mosaic (section visual crop above each editable dark-glass feature card). Hero/gallery → product cutout (`contain`), `kind: "section"` crops → feature image tiles (`cover`), video assets excluded from static placement.
- `ProductAsset.bgRemoved` is a required boolean. Live extraction normalizes it: assets under `/generated/removebg/` or named `(bg removed)` are `true`, section/video/upload assets are `false`. Image assets also carry `caption` and `semanticGroup`; deterministic metadata is always generated and, when `OPENAI_API_KEY` exists, OpenAI refines caption/group strings so similar photos share a group. `ff-cutout-image` sizing splits by `bgRemoved`: remove.bg cutouts use `fitImageToBox()` (`min(boxW/naturalW, boxH/naturalH)` — shorter dimension wins, no forced full width); all other assets use `fitImageFullWidth()` (`width = boxW`, height scales proportionally up to `boxH`). Template slots must preserve the asset's actual `bgRemoved`/caption/group values. Explicit `shape.props.w/h` box sizing avoids broken `%` max-height under tldraw zoom transforms.
- `ff-glass-card` tones: dark tones `tile`, `ink`, `frost`, `clear`, `accent`; Apple infographic light tones `paper` and `orange`. Titles use `currentColor` so light tiles can render dark text while dark tiles render white text. Copy tiles center-align their text horizontally and vertically. Card `body` text is kept in shape props and editable in the Inspector, but canvas rendering hides it by default unless `showBody` is explicitly true. `ff-glass-text` renders white display text; implementation-only `cutout` badges are not shown in export artwork.
- Editor asset behavior: source image assets can be dragged from the left panel onto the tldraw stage and are placed at the drop point via `editor.screenToPage`; clicking an asset remains an explicit copy-to-canvas action. Drag payloads are cleared to AdFrame's custom MIME type, thumbnail `<img>` nodes are `draggable={false}`, and stage drop handlers run in capture phase with `stopPropagation()` so tldraw/browser default URL drops do not create caption/link cards. Existing canvas image shapes also set their internal `<img>` to `draggable={false}` and prevent native `dragstart`, so dragging a placed asset moves the tldraw shape instead of copying/dropping its image URL. Newly created asset/feature shapes are immediately selected so the inspector reflects them.
- Canvas image assets (`ff-cutout-image`) are freely resizable in width and height; `CutoutImageShapeUtil.isAspectRatioLocked()` must stay `false`. Internal image rendering still preserves the source image's visual fit within the user-sized box, but the tldraw box itself should not force a fixed aspect ratio.
- Editor toolbar now exposes explicit icon buttons for `Undo` (`editor.undo()`) and `Redo` (`editor.redo()`), both with `aria-label`s; the interaction regression test asserts both hooks.
- Agents SDK layout review now lives server-side in `app-web/lib/agents/layout-review.ts` and `app-web/app/api/layout-review/route.ts`. The editor `Review layout` button captures the active tldraw frame PNG (`editor.toImage`), serializes frame-child shape metadata, assets, features, export preset, and position preset, then posts to `/api/layout-review`. The server validates with Zod, runs deterministic layout checks always, and uses `@openai/agents` with local tools (`measure_layout`, `rank_assets_for_slot`, `propose_grid_reflow`) when `OPENAI_API_KEY` exists. If the key is missing or the SDK call fails, it returns deterministic findings/patches only.
- Layout-review patches are intentionally conservative and reversible through tldraw undo/redo: `moveResize`, `swapAsset` (never video assets), `updateProps`, and `bringToFront`, sanitized to active-frame shapes only. Current deterministic checks cover overflow, substantial overlap, Apple hero centering, safe-margin signals, plus dense Apple board metrics: at least 9 non-hero tiles, non-hero coverage, grid occupancy, top/middle/bottom + left/center/right zone coverage, and hero-adjacent tile wrapping. Asset ranking uses current asset metadata (`id`, `name`, `alt`, `kind`, `bgRemoved`, `mediaType`) so future generated captions can improve scoring without changing the editor contract.
- Carousel Split was reimplemented from a preview modal into an editable multi-frame flow, but is hidden for hackathon focus behind `ENABLE_CAROUSEL_FEATURES = false` in `app-web/app/editor/page.tsx`. Keep the code so it can be revived later by flipping that flag. Hidden implementation: `composeCarouselSplitTemplate()` in `app-web/lib/tldraw/templates.ts` generates 3–5 real tldraw carousel frames (`carousel-slide-frame-*`, 1080x1350 feed format) placed beside the master frame; each slide has `ff-canvas-bg`, slide badge, editable headline text, editable image, editable copy card, and footer badge. When enabled, ZIP export prefers existing editable carousel frames via `downloadCarouselFramesZip()`; the old product-derived canvas renderer remains a fallback when no carousel frames exist. Current hackathon UI shows only main-canvas controls/exports (PNG/JPG/PDF), no Carousel Split button or Carousel ZIP button.
- Editor canvas sizing: the glass `.editor-stage` is shaped to the active export aspect ratio via `--stage-ratio` (inline from `preset.ratio`, e.g. `4 / 5`) with `aspect-ratio` + fixed height + `width: auto` + centered, so the frame fills the stage (no empty black box) and the stage stays compact. The middle editor `<section>` is centered with `maxWidth: 560` and the title uses `text-2xl`, so the "Glass canvas" header panel shrinks together with the canvas. The old `COMPACT_CANVAS_ZOOM` camera hack was removed in favor of this; `zoomToFit()` now fills the aspect-shaped stage directly.
- Manual feature placement uses the same `tile`/`accent` tones as the default style. Inspector tone options include `tile` and `accent`, and selected layers can be duplicated, brought to front, or sent to back.
- In-browser local background removal helper: `app-web/lib/segmentation.ts` using `@imgly/background-removal`.
- Editor background removal now calls server `/api/remove-background` first (remove.bg), then falls back to local `@imgly/background-removal` if remove.bg is unavailable or the source is unsupported.
- Source Images in Analysis/Editor are populated with real background-removed Samsung TV assets after extraction; repeated extraction reuses cached files instead of spending remove.bg credits again. Generated cache folders are URL/SKU-specific (`/generated/removebg/<sku>/`, `/generated/sections/<sku>/`, `/generated/videos/<sku>/`) so importing a new Samsung PDP (for example S90H `QN77S90HAEXZA`) cannot reuse the previous S90F source images. Section crop filenames also include an 8-character source URL hash to prevent stale same-index cache collisions within a SKU as the scraper learns new image candidates.
- Export helper: `app-web/lib/export.ts` using `editor.toImage`, `jspdf`, and `jszip`.
- Carousel ZIP export renders one PNG per feature slide via an offscreen canvas (`downloadCarouselSlidesZip`) rather than re-exporting the master frame.
- `/editor` requires the analysis approval flag in `localStorage`; direct editor navigation redirects to `/analysis`.
